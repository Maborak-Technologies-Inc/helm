## Chart Architecture (amazon-watcher-stack)

### Values Hierarchy

```
values.yaml
├── global.*           # domain, HPA toggle, shared storage PVC (enabled/accessModes/size/storageClassName), ArgoCD metadata
├── database.*         # PostgreSQL image, storage, credentials, NetworkPolicy
├── backend.*          # image, resources, env (APT_BACKEND_*), CLI, CronJob, Rollout, HPA
├── maborak.*          # utility pod image and resources
├── ui.*               # image, resources, env (VITE_*), Rollout
├── screenshot.*       # image, resources, Rollout
├── ingress.*          # routing feature flag
├── istio.*            # service mesh; istio.telemetry.* controls Istio Telemetry CRDs
├── secrets.*          # JWT secret (if empty, auto-generated by pre-install Job)
├── serviceAccount.*   # RBAC creation
├── argocd.*           # annotations injected into resource metadata
├── podSecurityContext.*  # global pod security context
└── securityContext.*  # global container security context
```

### Key Conventions

- Computed env vars (`DATABASE_URL`, `SCREENSHOT_SERVICE_URL`, `DOMAIN_UI`) are injected in templates, NOT in values
- Env vars carry component prefix: `APT_BACKEND_*`, `VITE_*`
- All application workloads use Argo Rollout CRD, not Deployment
- Feature gates: `istio.enabled`, `ingress.enabled`, `global.hpa.enabled`
- Storage class uses `lookup` function to detect `nfs-client` at render time

## Evaluation Criteria

### Chart Structure

| Concern | Standard |
|---------|---------|
| `Chart.yaml` | `apiVersion: v2`, semver version, maintainers listed |
| `values.yaml` | All fields documented with comments, sensible defaults, no secrets |
| `_helpers.tpl` | DRY helpers for repeated patterns, namespaced with chart name |
| `templates/NOTES.txt` | Post-install instructions for the user |
| `values.schema.json` | JSON Schema for values validation (recommended) |
| `.helmignore` | Excludes `.git`, `*.md` (except NOTES.txt), test fixtures |

### Template Quality

| Rule | Rationale |
|------|-----------|
| Use `include` (not `template`) for helpers | `include` captures output as string, works in pipelines |
| Always quote `.Values.*` strings with `{{ .Values.x \| quote }}` | Prevents YAML type coercion (`"true"` → bool) |
| Use `default` for optional values | `{{ .Values.x \| default "fallback" }}` |
| Guard with `if` before `range` on optional lists | Empty list renders nothing; nil list panics |
| Use `with` to scope context | Reduces `.Values.deeply.nested.path` repetition |
| Indent helpers correctly | `{{ include "helper" . \| nindent N }}` not `indent` |
| Use `required` for mandatory values | `{{ required "database.password is required" .Values.database.password }}` |
| Name all templates with chart prefix | `amazon-watcher-stack.x` not just `x` — prevents collisions |

### Values Design

| Principle | Guidance |
|-----------|---------|
| **Sensible defaults** | Chart should render valid manifests with zero overrides |
| **No secrets in defaults** | Passwords, keys use `""` or `required` — never placeholder strings |
| **Flat where possible** | Prefer `backend.replicaCount` over `backend.scaling.replicas.count` |
| **Consistent naming** | camelCase throughout, match Kubernetes field names where applicable |
| **Feature flags as booleans** | `istio.enabled: false` not `istio: "disabled"` |
| **Image as structured object** | `image: { repository, tag, pullPolicy }` — standard convention |

### Computed Values and Lookups

The `lookup` function in `_helpers.tpl` (for detecting `nfs-client` StorageClass) has implications:
- `lookup` returns empty during `helm template` (no cluster connection)
- Charts using `lookup` must have a fallback for dry-run/CI scenarios
- Document this behavior in values.yaml comments

### Env Variable Injection Pattern

This chart uses a sophisticated pattern:
1. Static env vars defined in `values.yaml` under component `.env` maps
2. Computed env vars injected directly in templates (DB URL, service URLs)
3. Helpers iterate the env map but skip computed vars to avoid duplicates
4. SHA256 checksum of env config forces pod restart on change

Validate that:
- Computed vars are excluded from the iteration list
- Checksum covers ALL config sources (not just env map)
- New env vars added to values.yaml are picked up by the helper

## Anti-Patterns — Flag Immediately

- **`template` instead of `include`** — output can't be piped, leading to indent bugs
- **Unquoted string values** — `"true"`, `"null"`, `"1.0"` silently become bool/null/float
- **No `default` or `required` on optional values** — nil pointer panic at render time
- **Deeply nested values with no `with` scope** — unreadable templates, easy to mistype paths
- **Hardcoded namespace in templates** — breaks when installed in a different namespace
- **`lookup` without fallback** — breaks `helm template` and CI dry-runs
- **Checksum not covering all config sources** — pods don't restart when config changes
- **Duplicate env var injection** — computed var also appears in values env map
- **Chart version not bumped on change** — published chart is stale or conflicts
- **No `.helmignore`** — packages include `.git`, docs, tests, bloating the chart
- **Using `toYaml` without `nindent`** — YAML indentation breaks silently
- **Secrets in values.yaml defaults** — even "example" secrets end up in Git history
